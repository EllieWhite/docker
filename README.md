В среде исполнения Node.js нет браузера, поэтому document, window не будут использованы.
- В Node.js есть встроенная система модулей - можно создавать другие файлы, подключать и кк-то связывать их. c помощью функции require: require('./module');
- Больше доступа к различным сущностям, например, можем вывести глобальные переменные, пути;

**Как работают модули**

    Есть 2 основных типа модулей Node.js:
        1. Встроенные модули в саму программу. Можно обратиться к модулю из любых программ, из любого места. 
        2. Сторонние модули, который устанавливаются с npm.

    Для разработки программы установим модули:
    npm i yargs - для чтения параметров и документации


**Нюансы использования __dirname и __filename**

    При изменении импортов и экспортов на модульность Node.js, глобальные константы не определяются. 
        Это связано с тем, что каждый модуль при импортах оборачивается в самовызывающуюся функцию.
        При использовании модульности необходимо определить глобальные переменные:
            import path from 'path'
            import { fileURLToPath } from 'url'

            const __filename = fileURLToPath(import.meta.url) // получаем название файла
            const __dirname = path.dirname(__filename)

            console.log(__filename)
            console.log(__dirname)

**СУБД по способу хранения**

    - Табличные - информация хранится в таблицах
    - Документоориентированные - в документах. Могут иметь разную длину полей, например, для телефона - блютуз, для пылесоса - провод.
    - Темпоральные (временные) - бд, которые хранят информацию, привязанную ко времени. Часто используют для хранения событий, произошедших в определенный момент времени
    
    Табличные и документоориентированные БД покрывают большинство потребностей для небольших и средних приложений.


**СУБД по способу обращения**

    - SQL - специальный язык запросов, который позволяет обращаться к базе данных. Работает среди нескольких БД: postgressql, mysql и т.д
    - NoSQL - кажда из БД предоставляет какой-то свой способ общения с не и получения данных. Часто можно встретить к-л способы, которые проще, чем сам SQL, но для каждой бд скорее всего нужно изучать свой способ взаимодействия с ней.
    

Mongo DB - одна из популярных документоориентированных БД без жесткой схемы.
    Её структура: документы => коллекции => коллекции сложены в БД, т.е внутри одной установленной программы (инсталляции) MDB
    могут быть несколько БД для разных приложений. Такое возможно для локального использования, но нельзя использовать в проде, т.к в этом случае разные БД будут делить между собой ресурсы.

    Способы установки MDB:
        1. Локальный. Из минусов-  неудобно переносить данные между локальным и тестовым сервером. Иначе необходимо создать новую бд, установить на сервер с приллжением и наполнить новыми даннымм.
        2. Воспользоваться готовым сервисом MDB. Один из них - MDB Атлас. Преимущество- бесплатный тарифный план. Из минусов- может понадобиться впн. Еще один минус бесплатного тарифа - его ресурсы переиспользуются между всеми бесплатными кластерами всех пользователей, под нагрузкой может работать не стабильно.
    
    Драйвер - специальная программа, которая умеет общаться с СУБД.

    ORM (Object-Relational Mapping) - технология, которая обеспечивает сопоставление объектов и классов в языке программирования с таблицами и структурами в реляционной базе данных, позволяя работать с данными в объектно-ориентированной парадигме.

    ODM (Object-Document Mapping) - технология, которая обеспечивает сопоставление объектов и классов в языке программирования с документами и коллекциями в документоориентированной базе данных, упрощая работу с неструктурированными данными. Одна из самых популярных - Mongoose.
    

Схема MONGOOSE - описание самой структуры данных. Позволяет валидировать данные перед отправкой. На уровне базы при этом не будет происходить проверки.
    Проверка происходит на уровне приложения. Модель позволяет сохранять даные, доставать и т.д. Модель - абстракция, с которой будем работать.

    СХЕМА:
        const NoteSchema = mongoose.Schema({
        title: {
            type: String,
            required: true
        }
        })

    МОДЕЛЬ:
        const Note = mongoose.model('Note', NoteSchema)
